# Product

## Register

brand

## Users

Visitors, collaborators, and technical peers use Scra Atlas as a project-index homepage. They arrive to understand what is live, what is being built, and where to find public work without scanning a conventional portfolio grid.

## Product Purpose

Scra Atlas presents personal projects as indexed signals inside a digital atlas interface. Success means the page feels memorable and precise while still making the project registry, project status, and GitHub link easy to find.

## Brand Personality

Quiet, technical, cinematic.

## Anti-references

The interface must not look like a NASA wallpaper, a generic galaxy background, a dense particle-network template, or a demoscene effect that competes with project information. Celestial imagery should not become the main thing visitors remember.

## Design Principles

- Treat the shader as the living field and the UI as the product.
- Make signal, coordinate, scan, and index language more important than literal stars or nebulae.
- Keep the center readable and let motion stay slow, low-frequency, and observatory-like.
- Use interaction as subtle measurement feedback, not spectacle.
- Preserve provenance and maintainability for all visual sources.

## Accessibility & Inclusion

Respect reduced-motion preferences across shader time, pointer response, scroll behavior, and CSS transitions. Maintain keyboard access for project coordinates, registry rows, external links, and the close control. Keep text contrast high on the dark field and avoid relying on color alone for project status.
