## 用户

Scra Atlas 首先服务于愿意探索一个开发者数字空间的访客，包括技术同行、潜在合作者和对个人项目感兴趣的人。招聘者可以从中理解经历与能力，但网站不以快速筛选简历为首要目标。

## 产品目的

Scra Atlas 是一个持续生长的个人技术档案，用来呈现项目、工程系统、AI 实验、技术文章和个人发展轨迹。

它不是传统作品集，也不是把简历换成黑色背景的网站。访问者首先应因独特的视觉与交互产生记忆，然后通过项目树、时间线和文章理解 Scra 构建过什么、如何思考以及正在探索什么。

成功意味着：

- 访客在短时间内形成明确印象，记住这是一个“正在运行的技术档案”。
- 特效与内容存在因果关系，扫描用于发现、展开用于理解、状态变化用于建立当前位置。
- 项目与文章可以持续增加，不要求重新设计全站。
- 任一视觉层失效或被关闭时，核心内容与导航仍然完整可用。

## 当前归档形态

根页的内容顺序固定为 **Hero → Timeline → Projects → Logs**。Current
Index 仍属于 Hero 的既有构图，不是额外的归档章节；Hero 的中文描述、入口和
视觉职责保持不变。

- **Timeline** 用已核实的发展节点让访客先理解时间脉络。
- **Projects** 用可选择的项目索引和系统树解释当前的工程方向。
- **Logs** 在同一连续页面中打开可读的工程笔记，而不是跳转到独立博客页。
- **Lab** 在本次迭代中延后：其源文件可以保留，但不进入根页或活动归档导航；
  旧的 `/lab` 路由安全地重定向到根页的 `/#index`。Current Index 中的
  Infrastructure Lab 条目暂时指向 Projects，直到 Lab 表面重新出现。

## 内容与编辑边界

作者维护的归档记录全部保留在本地语言包：`src/content/zh/` 与
`src/content/en/`。每个语言包都有 `site.json`、`timeline.json`、
`projects.json`、`logs.json` 与 `logs/*.md`。服务端内容加载器校验 JSON 记录、
时间线排序、唯一标识和日志文件名，再将可序列化的类型化数据交给根页。

日志 Markdown 只能通过当前语言包 `logs.json` 中已登记的简单 `.md` 文件名读取，
并且只会在对应的 `logs/` 目录内解析；浏览器不传递文件路径，也不会选择任意本地
文件。Markdown 先转换为安全的结构化块，再由语义 DOM 渲染，不注入原始 HTML。
`HomeExperience` 只负责按顺序组合当前语言内容，不解析 Markdown，也不访问文件系统。
新增经历、项目或笔记应通过内容记录完成，而不是修改特效或归档 UI。中文包可保留
技术英文，英文包会在加载时拒绝任何中文字符，避免语言混入。

语言切换使用两个静态入口：`/` 为中文，`/en` 为英文。`ZH / EN` 控制使用完整页面
跳转，因此英文页面只接收英文内容包，不会保留中文的客户端序列化数据，也不需要
运行时内容请求。

## 品牌个性

三个核心词：**精确、动态、有作者性**。

- 精确：排版、状态、层级和反馈都有明确目的。
- 动态：网站像一个活着的索引，不像静态宣传册。
- 有作者性：视觉选择来自 Scra 的项目与经历，不套用通用开发者模板。

## 反例

Scra Atlas 不应成为：

- 星球、星云、黑洞、虫洞或天体旅行体验。
- 依赖发光粒子、霓虹、扫描线堆砌出来的廉价赛博朋克界面。
- 满屏终端窗口、假日志和无意义状态码的开发者装扮。
- 由相同圆角卡片组成的普通作品集。
- The Content Architecture 或任何参考站的品牌、版式、文案、颜色、代码或动画复刻。
- 为了展示 Canvas、WebGL 或 shader 而存在的技术 demo。
- 为了避开上述反例而退化成没有主视觉、只有黑白文字的保守页面。

## 产品原则

1. **先被看见，再被理解。** 视觉体验负责吸引，内容结构负责留下理由。
2. **每页一个主秀时刻。** 每个主要页面拥有一个不可互换的核心效果，其他动效为它服务。
3. **效果表达信息状态。** 动画必须对应定位、加载、选择、展开、归档或转移等真实状态。
4. **内容与效果解耦。** 添加文章、项目或经历时，不修改渲染引擎；替换视觉场景时，不修改内容数据。
5. **高质量优先于技术纯度。** DOM、SVG、Canvas、WebGL、shader 或图片都可采用，以观感、性能和维护成本决定。
6. **失效时仍然成立。** 禁用脚本、减少动画、触摸输入或渲染失败时，网站仍是完整的个人技术档案。

## 无障碍与包容性

- 以 WCAG 2.2 AA 为基础目标。
- 完整支持 `prefers-reduced-motion`，并提供可发现的特效强度控制。
- 主要信息不依赖 hover、颜色、随机字符稳定过程或连续动画才能读取。
- 所有项目树操作、导航和展开行为支持键盘与触摸。
- 不使用高频闪烁、突然缩放、强烈视差或无法停止的高速运动。
