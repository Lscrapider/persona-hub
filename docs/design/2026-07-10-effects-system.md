# Scra Atlas 特效系统

**状态：当前有效，具体参数允许在视觉原型后校准**  
**更新日期：2026-07-10**

## 目标

特效系统负责让 Scra Atlas 成为一个可被记住、可被探索的数字空间。它不是装饰插件集合，而是一套表达“定位、读取、展开、切换、归档”状态的视觉语言。

技术实现不预设。DOM、CSS、SVG、Canvas、WebGL、shader、图片序列或第三方动画运行时都可以参与；每个场景必须用最合适且能被维护的方案，而不是全站统一押注某一种技术。

## 三层结构

### 1. Signature Scenes：页面主秀

每个主要页面最多一个主秀场景。主秀可以是 feature-specific，不要求为了复用而削弱艺术方向。

| 页面 | 主秀场景 | 要表达的状态 |
| --- | --- | --- |
| Home | Kinetic Type Field | 身份被识别，档案开始运行 |
| Projects | Project Tree Navigator | 系统被定位、展开和读取 |
| Project Detail | Architecture / Evidence Scene | 模块关系和数据流被解释 |
| Blog Index | Knowledge Field | 笔记被聚合、筛选和聚焦 |
| Timeline | Reading Head | 时间记录被逐段扫描 |
| Lab | Isolated Experiment | 独立实验进入运行状态 |

### 2. Motion Grammar：共享动作语法

共享语法让不同页面属于同一个系统，但不能机械地在每个 section 重复。

- `Resolve`：字符、路径或状态从不确定收敛到可读结果。
- `Locate`：光标、路径、下划线或框选定位当前目标。
- `Open`：节点、段落或面板按结构展开。
- `Transfer`：当前模块退出，下一模块接管状态。
- `Archive`：内容离场时保留路径或索引痕迹。
- `Focus`：背景密度下降，主要信息取得前景权重。

### 3. Atmosphere：氛围层

氛围层包括低对比网格、静态噪声、坐标痕迹、弱扫描纹理、极少量数据点或光标附近的局部响应。

氛围层必须满足：

- 去掉后页面仍有完整构图。
- 不持续争夺注意力。
- 不制造“太空”“雷达”“黑客终端”的直接联想。
- 低性能或减少动画模式下可以整层关闭。

## 核心效果规格

### Entry Resolve

用途：首次访问时建立仪式感。

- 最终标题与进入控制必须从第一帧就存在于可访问结构中。
- 完整编排建议在 700–1400 ms 内结束，不做假进度条。
- 回访、直达子路由和减少动画模式不重复完整片头。
- 用户可以立即跳过或操作导航。
- 允许短系统文案，但最多三条，且必须描述真实状态。
- 内联入口门控必须提供不依赖 React hydration 的点击与超时解锁；客户端场景加载失败时不能留下永久遮罩。

### Scramble / Character Resolve

用途：短英文标签、导航状态、按钮或系统状态发生变化时的字符收敛。

- 只处理适合随机字符的短拉丁文本；中文正文不做乱码解码。
- 标点、空格和最终文本长度保持稳定，避免布局抖动。
- 单次建议 300–700 ms；同一标签在一次页面生命周期内不循环播放。
- hover 不持续触发整段 scramble，最多在首次聚焦时触发一次轻量版本。
- reduced 模式使用 120–180 ms 透明度或直接显示最终文本。

### Kinetic Type Field

用途：首页视觉签名。

- 由短词、路径文字、尺度变化和密度层级构成，不等同于一个匀速旋转圆环。
- 前景必须有稳定标题、签名和导航；背景排版场不得妨碍阅读。
- 静止时也必须是一张成立的海报式构图。
- 指针只轻微改变局部速度、方向或焦点，不能形成追随鼠标的廉价粒子尾巴。
- 滚动离开首屏时场景收束或归档，不在后台永久满负荷运行。
- 具体采用 SVG 路径、DOM 排版、Canvas 或 WebGL，必须通过视觉原型比较后决定。
- 首页首版采用语义 DOM + SVG 路径文字实现；渲染层必须可替换，首版不为了技术炫耀引入 WebGL。
- 批准构图为骨白开放信息面与黑色文字场的非对称切入，不使用多个黑色面板把文字分别框住。

### Project Tree Navigator

用途：Projects 页的核心交互与信息结构。

- 节点展开 220–420 ms，路径与详情变化保持同一因果顺序。
- 当前节点、祖先路径和详情标题必须同步，不出现视觉高亮与内容错位。
- hover 显示补充 metadata；focus 和点击提供等价能力。
- 树的连线可以有方向性运动，但只在状态变化时激活，不持续流动。
- 大型项目树需要虚拟化或分段加载时，不能破坏 URL 定位和键盘顺序。
- reduced 模式保留展开层级与当前路径，取消连续线条运动。

### Reading Head

用途：Timeline 的滚动叙事。

- 原生滚动位置决定当前记录，不使用强制吸附或滚动劫持。
- 日期、事件、轨道游标和背景密度在同一状态源下更新。
- 状态切换建议 320–600 ms，快速滚动时直接追上最新状态，不排队播放旧动画。
- 文字在动画失败时默认可见；游标只是辅助，不是唯一信息载体。
- 移动端使用单列时间轨迹，取消依赖横向空间的指针或多列错位。

### Rule / Path Draw

用途：连接、定位与边界形成。

- 线条必须表达层级、方向或状态，不作为随处可见的装饰。
- 建议 240–520 ms，使用 clip、mask、stroke 或 transform 等不会引起布局变化的方式。
- 线条完成后保持稳定，不反复擦除重画。

### Metadata Reveal

用途：补充技术栈、状态、路径和时间。

- 主要内容始终可见，metadata 才可以按需揭示。
- hover、focus-visible 和触摸展开具有等价结果。
- 建议 160–280 ms，不使用大范围模糊和发光。

### System Transition

用途：页面或主要模块之间的状态转移。

- 视觉语义为关闭当前记录、转移路径、激活下一记录。
- 总时长建议 450–900 ms；导航不能等动画结束后才开始。
- 转场失败或浏览器不支持时回退到正常路由切换。
- 不使用虫洞、全屏光束、巨大 zoom 或长时间黑屏。

## 动作参数

共享参数用于建立节奏，不要求所有效果使用同一时长。

```text
instant   100–160 ms   状态确认、按压反馈
fast      160–280 ms   hover、metadata、短焦点移动
base      280–480 ms   节点展开、路径绘制、内容切换
scene     480–900 ms   页面场景进入与转移
ritual    700–1400 ms  首次访问编排上限
```

- 进入使用明确的 ease-out，离场可以更短。
- 不使用 bounce、elastic 或卡通回弹。
- 不通过动画修改大量布局属性；优先 transform、opacity、clip、mask 和合适的渲染层。
- 同一视口最多存在一个主导动画和一个低强度连续背景。

## 触发与重播

| 触发 | 允许行为 | 禁止行为 |
| --- | --- | --- |
| 页面首次进入 | 完整主秀编排 | 每次返回视口都重播片头 |
| 滚动进入 | 与内容结构对应的状态推进 | 所有 section 统一 fade-up |
| hover / focus | 局部定位、metadata、短字符变化 | 持续乱码、闪烁、追逐光效 |
| 点击 / 键盘操作 | 展开、选择、转移 | 先播放无意义动画再响应 |
| 空闲 | 极慢背景呼吸 | 高频随机变化或不可停止运动 |
| 页面隐藏 / 离屏 | 暂停连续渲染 | 后台持续 rAF、Canvas 或 WebGL |

## 效果强度模式

系统提供三种运行状态：

- `full`：完整主秀、输入响应和氛围层。
- `reduced`：保留状态变化，取消连续运动、视差和字符扰动。
- `static`：展示设计好的最终帧，所有内容与操作仍可用。

初始模式参考系统减少动画设置、输入类型和运行表现；用户手动选择始终优先，并在本地保存。

## 审美检查

实现任何效果前必须回答：

1. 没有动画时，这一帧是否仍然是一张成立的界面？
2. 动画表达了什么真实状态？
3. 为什么它属于这个页面，而不是可以随便复制到另一个页面？
4. 它是否与当前主秀争夺注意力？
5. 去掉发光、粒子和噪声后，设计是否仍然有辨识度？

无法回答时，不实现该效果。

## 第一阶段建议

第一阶段优先验证体验而不是一次完成所有页面：

1. Home 的 Kinetic Type Field 静态构图与完整编排。
2. Projects 的树结构、节点状态和详情联动。
3. 共享的字符收敛、路径绘制、metadata 与转场语法。
4. full、reduced、static 三种状态的可行性。

Timeline、Blog 和 Lab 在上述语言稳定后接入，避免同时开发多套未经验证的视觉系统。
