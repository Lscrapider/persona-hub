# Scra Atlas：特效主导体验设计

**状态：当前有效**  
**更新日期：2026-07-10**

## 一句话方向

Scra Atlas 是一个可探索的个人技术档案。它用高密度排版、项目树、状态变化和滚动编排，让访客感觉自己正在读取一个持续生长的系统。

## 体验优先级

本项目的首要卖点是视觉与交互体验。内容不是动效后面的占位物，而是动效的素材和原因。

优先级如下：

1. 建立独特、可记忆的第一印象。
2. 让访问者愿意继续探索项目、经历和文章。
3. 保证内容可读、导航清楚、移动端可用。
4. 在不破坏前三项的前提下优化实现复杂度。

## 信息架构

网站采用独立路由与共享外壳，不把所有内容强行塞进一个超长首页。

```text
/                     入口仪式、个人签名、当前索引
/projects             项目档案树
/projects/[slug]      项目说明、能力、架构与证据
/blog                 Markdown / MDX 技术日志索引
/blog/[slug]          文章阅读页
/timeline             经历与成长轨迹
/lab                  有边界的视觉和工程实验
```

`Entry` 是首页上的首次访问状态，不是阻断直达链接的独立路由。联系入口属于全站外壳或页尾，不需要单独制造一个空洞页面。

## 页面角色

### Home：访问票与个人签名

Home 不承担复杂功能。它需要在一个视口内完成三件事：

- 告诉访客这是 `Scra Atlas`。
- 用一句作者签名解释这是一个怎样的空间。
- 通过一个强主视觉建立记忆，并给出进入 Projects、Blog、Timeline 的明确入口。

主视觉不是星体，也不限定为简单文字圆环。它是一组围绕 `Scra Atlas`、项目、构建、研究和部署语汇形成的动态排版场。文字可以沿路径运动、改变层级、响应指针或在滚动时重新编排，但必须保留清楚的前景标题和导航。

### Projects：项目树

Projects 是网站的核心功能与核心特效页面。项目不是同尺寸卡片，而是一个可读取的档案树。

```text
projects
├── financial-management-ai
│   ├── ingestion
│   ├── ocr
│   ├── agent-runtime
│   └── reports
├── urban-sidequest
│   ├── poi
│   ├── route
│   └── recommendation
└── experiments
```

树节点代表真实信息结构。选择节点时，详情区、路径、状态和相关连线同步变化。展开与高亮必须支持键盘、触摸和直达链接。隐藏的 metadata 只能是补充信息，不能是理解项目所必需的唯一信息。

### Project Detail：项目证据

项目详情页回答四个问题：为什么做、系统如何工作、本人负责什么、现在处于什么状态。

允许使用架构图、流程图、代码摘录、截图、演示媒体和真实指标。装饰性终端不是默认答案。每个项目根据自身内容选择最合适的证据形式。

### Blog：工程日志

Blog 用 Markdown 或 MDX 管理。索引页可以使用密集文字场、标签聚合和字符聚焦效果；文章页则以阅读质量为主，只保留低强度的进度、索引和代码交互。

特效不能导致正文漂移、滚动不稳定或代码无法选择复制。

### Timeline：个人轨迹

Timeline 用时间推进而不是普通卡片列表。当前年份、阶段或事件与滚动位置建立清晰映射。背景文字与历史记录可以形成密度变化，但经历正文必须始终可读。

具体年份和经历属于内容数据，不写死在视觉组件或本设计文件中。

### Lab：有边界的实验

Lab 用于放置不适合成为全站基础设施的视觉实验、交互原型和工程小工具。实验可以大胆，但必须与全站导航、性能和错误隔离。

## 视觉语言

### 场景描述

访客在较暗环境中打开网站，像进入一张正在被读取和重新索引的技术工作台。界面应有强烈排版构图、精确的边界和有节奏的状态变化，而不是发光仪表盘。

### 设计材料

- 当前批准的首页配色是骨白、近黑、冷银灰和少量陶土橙。骨白与近黑承担大面积结构，陶土橙只用于活动路径、焦点和关键状态。
- 等宽字体用于状态、路径、日期、代码和元数据，不承担全部正文。
- 展示字体、正文字体和等宽字体最多三套，中文和英文必须各自可读。
- 细线、路径、裁切、遮罩、字符密度和负空间是主要构图材料。
- 噪声、网格、扫描痕迹和弱粒子只能作为氛围层，不能成为视觉主题。
- 普通标题、正文、导航和 metadata 依靠排版与留白组织，不使用矩形框逐块包围。边界只属于真正有范围或可操作状态的元素。

首页实现以 [批准的视觉北星](references/2026-07-10-home-north-star.png) 为构图依据，以 [调色与字型氛围板](references/2026-07-10-home-palette.png) 为颜色关系依据。视觉稿是方向约束，不是需要逐像素复制的截图。

## 全站体验节奏

```text
识别身份 → 建立主视觉 → 定位内容 → 展开节点 → 阅读证据 → 归档并转移
```

- 首次进入可以有短仪式，但不能用假加载拖延内容。
- 页面进入不是统一 fade-in。不同内容使用与其含义匹配的出现方式。
- 页面切换表现为状态转移，例如定位、关闭、归档、载入和激活；文案必须少而准确。
- 每个视口同一时刻只有一个主要注意力中心。
- 不劫持原生滚动，不让用户等待无法跳过的片头。

## 响应式原则

- 桌面端可以使用分屏、sticky、路径和指针响应。
- 触摸设备用点击、聚焦和滚动状态替代 hover，不模拟假光标。
- 移动端减少同时运动的层数，不缩小桌面构图硬塞进屏幕。
- 320 px 宽度仍可访问全部主内容；装饰层可以消失，信息层不能消失。

## 明确不做

- 不回到宇宙、天体和穿梭叙事。
- 不以 Canvas 或 WebGL 的存在本身作为卖点。
- 不在所有页面重复同一个文字解码、扫描线或卡片进入动画。
- 不用大量状态码伪装内容密度。
- 不把参考站复制成个人版。
